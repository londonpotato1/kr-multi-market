## 0. Conventions

### Atomic Commits
- One commit per task, conventional commits style: `feat(server/hl): add metaAndAssetCtxs adapter (T1.4)`
- Tag `phase-N-complete` after each phase QA gate passes
- `gitleaks protect --staged` pre-push hook (configured Phase 0)
- Never amend pushed commits; create follow-up `fix:` commits

### TDD Policy
- **Pure-logic modules → tests FIRST**: HL schema, GDR guard, session matrix, z-score
- **Network adapters → tests AFTER**, gated by `INTEGRATION=1` env var
- **UI → manual QA + Vitest snapshot only where stable**

### Project layout decision (Sisyphus override of plan agent)
- **Single-package** (NOT monorepo). Root = Vite. `server/` = Express. No `pnpm-workspace.yaml`.
- Rationale: simpler for single dev, Vite already scaffolded at root, monorepo restructure = unnecessary churn.

### Delegation Cheat Sheet
| Need | Category |
|---|---|
| Pure TS logic, schemas, math | `ultrabrain` |
| Adapters, glue code | `deep` |
| Trivial edits, single-file fixes | `quick` |
| UI components | `visual-engineering` |

| Subagent | Use |
|---|---|
| `explore` | Find existing patterns |
| `librarian` | External vendor docs |
| `oracle` | QA gates, complex review |
| `code-reviewer` | End-of-phase review |
| `quick-validator` | Smoke checks |

## 3. Per-Wave Execution Recipe

For each wave, dispatch parallel-eligible tasks via `task()` tool with:

```
task(
  category="<delegate_to>",
  load_skills=[<load_skills>],
  description="<task id and short title>",
  prompt="...read PLAN.md sections + IMPLEMENTATION_PLAN.md task spec + commit atomically...",
  run_in_background=true  # for parallel; false for QA gates
)
```

**On any QA failure → halt the wave, do not proceed**.
