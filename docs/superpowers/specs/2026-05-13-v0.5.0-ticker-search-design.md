## 1. Context & Motivation

v0.4.2 까지 대시보드는 7개 정적 ticker (samsung, skhynix, hyundai, kospi200f, ewy, sp500, nq + usdkrw) 만 표시. 사용자가 임의 종목 (예: AAPL, MSFT, 카카오, NVDA) 모니터하려면 코드 수정 필요.

**사용자 요청**:
> 원하는 ticker 검색하면 가격 모니터 추가 할 수 있도록 하고싶어.
> 신규 티커 검색의 경우 상장되어 있는 거래소를 순차적으로 여러 단계층으로 검색.
> Tier 1: 한국 증권거래소, 미국 나스닥. Tier 2: Binance, OKX 같은 CEX. Tier 3: Hyperliquid.
> 코인 말고 주식에 한해서.

### Goals

1. **검색 input** — 사용자가 종목명 (한글/영문) 또는 ticker symbol 입력
2. **3-Tier resolver** — Tier 1 (KRX/NASDAQ) → Tier 2 (CEX) → Tier 3 (HL xyz) 순차 fallback, 첫 hit 에서 stop
3. **Watchlist 영구 저장** — localStorage 에 사용자 종목 list, 새로고침 후 유지
4. **별도 UI section** — "📌 관심 종목" 별개 grid, 정적 카드와 분리
5. **삭제 기능** — 카드 우상단 X 버튼

### Non-Goals (사용자 명시 + 카파시 결정)

- ❌ 동기화 (multi-device, server stateful) — localStorage only
- ❌ 사용자 인증 / 다중 watchlist
- ❌ 차트, 뉴스, 재무 데이터
- ❌ 알림 (가격 alert)
- ❌ Drag-and-drop 정렬
- ❌ 코인 ticker (BTC/ETH 등) — 사용자 명시 "주식만"
- ❌ OKX, KuCoin 등 추가 CEX — v0.6 backlog
- ❌ 종목명 fuzzy match ("Aple" → AAPL) — exact + prefix only
- ❌ Hyperliquid main dex (crypto) — xyz dex (stock) only
- ❌ FX/Forex ticker (EURUSD)
- ❌ 검색 history, 자동완성 (autocomplete)
- ❌ Watchlist export/import

```
┌─────────────────────────────────────────────┐
│ Header (코스피 24시간 트래커)               │

├─────────────────────────────────────────────┤

├─────────────────────────────────────────────┤
│ 📌 관심 종목                                │ ← Q8: WatchlistSection
│ [WatchlistCard × N] + 우상단 [X] (Q11)      │

└─────────────────────────────────────────────┘
```

### 3.2 3-Tier Resolver (server-side, `/api/search` 단일 endpoint)

resolver 는 **server-side** 에 위치 (CORS, secret 보호, source 인프라 재사용). client 는 `GET /api/search?q=...` 호출 후 결과 dropdown 표시.

```
입력: GET /api/search?q=AAPL
  │
  ├─ Tier 1: KRX/NASDAQ (외부 search API)
  │   ├─ Hangul detect → searchNaver(q) → KRX 결과 (top 5)
  │   └─ 영문 → searchFinnhub(q) → NASDAQ/NYSE 결과 (top 5, type='Common Stock'|'ETF' filter)
  │   1+ hit → response = { tier: 1, results: [...] } → STOP
  │
  ├─ Tier 2: CEX probe (Binance/Bybit/Bitget) — Tier 1 결과 0개일 때만
  │   ├─ ticker + "USDT" suffix 자동 (AAPL → AAPLUSDT)
  │   ├─ fetchBinanceFutures(['AAPLUSDT']) + fetchBybitLinear(['AAPLUSDT']) + fetchBitgetFutures(['AAPLUSDT']) 병렬
  │   │  (v0.4.2 기존 source 함수 재사용 — 추가 endpoint X)
  │   ├─ 응답 ok && data 가 1+ valid PricePoint → hit
  │   └─ 우선순위 binance > bybit > bitget 으로 first source 채택 → response = { tier: 2, results: [{source, symbol, label}] } → STOP
  │
  └─ Tier 3: HL xyz universe — Tier 2 도 0개일 때
      ├─ getLatestMids() 에서 `xyz:${TICKER}` key 존재 검사 (v0.4.1 hyperliquid-ws.ts)
      └─ hit → response = { tier: 3, results: [{source:'hyperliquid', symbol:`xyz_${TICKER}`, label:TICKER}] } → STOP

모든 tier miss → response = { tier: null, results: [], reason: 'not_found' }
```

**Dropdown 동적 tier**: dropdown 결과 = 현재 hit 된 tier 의 결과만. Tier 1 hit 면 Tier 2/3 미조회. Tier 1 miss 시 Tier 2 결과 (1개) 만 표시. UX 흐름 = Q4 (first-match-wins) 와 일관.

**Frontend `ticker-resolver.ts` 의 책임 축소**: 검색어 → `/api/search` HTTP 호출 + 응답 dropdown 으로 전달만. 3-Tier 분기 server-side.

**카파시 가드**: server `/api/search` 안 inline 3-tier (if/else 단일 함수). Tier 별 클래스/Strategy 패턴 X. 단일 endpoint, 단일 함수.

### 3.3 Data Flow

#### 검색 → 추가
```
SearchBar Enter
  → GET /api/search?q=Apple
  → [server] Tier 1 (Hangul detect → Naver / 영문 → Finnhub /search)
     → Tier 1 miss 시 Tier 2 (CEX probe via 기존 fetchX 함수)
     → Tier 2 miss 시 Tier 3 (HL WS state lookup)
  → response { tier, results }
  → SearchDropdown 표시 (top 5 or 1)
  → 사용자 클릭 → entry = results[i]
  → useWatchlist.add(entry)
  → localStorage 갱신 + SWR mutate
  → WatchlistSection 카드 즉시 렌더 (loading)
  → 다음 SWR poll (2s) 후 가격 표시
```

#### 가격 fetch (매 2s SWR poll)
```
useWatchlist.entries → SWR key 구성
  → GET /api/prices?watchlist=finnhub:AAPL,naver:005930,binance:AAPLUSDT
  → server: parseWatchlist + 동적 source-별 symbols 합집합
  → source-level singleFlight cache 흡수
  → 정적 SYMBOL_TO_TICKER + 동적 watchlist mapping 통합
  → tickers 응답 (정적 + 동적)
  → WatchlistCard 가 IndexCompactCard 패턴으로 표시
```

#### 삭제
```
WatchlistCard 우상단 X 클릭
  → useWatchlist.remove(key)
  → localStorage 갱신 + SWR mutate
  → 카드 즉시 사라짐
  → 다음 SWR poll 부터 server query 에서 제외
```

## 5. localStorage Schema

**Key**: `kr-multi-market.watchlist.v1`

```json
[
  { "key": "aapl", "source": "finnhub", "symbol": "AAPL", "label": "Apple Inc.", "tier": 1 },
  { "key": "kakao", "source": "naver", "symbol": "035720", "label": "카카오", "tier": 1 },
  { "key": "tsla-perp", "source": "binance", "symbol": "TSLAUSDT", "label": "Tesla (Binance)", "tier": 2 }
]
```

### Key 충돌 처리

- 사용자가 정적 ticker (예: "EWY", "QQQ", "삼성전자") 검색 시 → 기존 정적 카드 있으니 **alert + 거부** ("이미 표시 중").
- 검색 key 가 다른 watchlist entry 와 충돌 시 → `-1`, `-2` suffix 자동 추가.

### 용량 제한

- 최대 50 entries (사용자 alert)
- localStorage 5MB cap 안전 마진

## 6. Error Handling + Edge Cases

| 시나리오 | 처리 |
|---|---|
| 검색 결과 0개 | dropdown "검색 결과 없음" 표시 |
| Finnhub 60/min quota 초과 (429) | "잠시 후 다시 시도" + 60s cooldown timer |
| Naver scrape 실패 | dropdown "한국 주식 검색 일시 장애" + Finnhub fallback (영문만) |
| 모든 Tier miss | dropdown "$ticker 어느 거래소에도 없음" + 추가 거부 |
| watchlist 중복 추가 (같은 key) | alert "이미 추가됨" + 거부 |
| 정적 ticker 와 충돌 | alert "이미 표시 중 (메인 카드)" + 거부 |
| localStorage 용량 초과 (5MB or 50 종목) | warning + 추가 거부 |
| server `/api/prices` watchlist source fail | 카드 표시 + 가격 "—" + sourceHealth 마킹 |
| 검색 ticker 가 코인 ("BTC", "ETH") | Finnhub filter (Common Stock + ETF type only) → dropdown 결과 없음 |
| localStorage 손상 / JSON parse fail | try/catch → 비우고 경고 toast |
| 새로고침 후 watchlist 복원 | useWatchlist init 시 localStorage 읽기 → SWR 자동 fetch |
| HL universe 확장 (새 ticker) | WS allMids 자동 catch → Tier 3 lookup 가능 |
| 모바일 < 640px | SearchBar full-width + 1-column watchlist grid |
