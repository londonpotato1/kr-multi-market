## 1. Context & Motivation

v0.3.0 도입 후 카드 위계는 명확해졌으나, 사용자 피드백:

1. **24h 변동률은 다국 시장 (미국/한국) 무관 — 한국 사용자에겐 직관 X.** "전일 종가 대비" 가 더 자연스러움.
2. **USDT 환산 가치 부재.** 코인 시장 사용자 다리 역할 부족.
3. **PRIMARY 가 항상 HL→KRW** — 국장 열려 있을 땐 KRX 실시간이 더 정확한 시장 가격임에도 보조 위치.
4. **초보자에게 영문 라벨 / 전문 용어 (PREMIUM, COOL, WATCH, z-score) 어려움.**

v0.4.0 의 미션 = "한국 주식 투자자가 처음 봐도 5초 안에 이해할 수 있는 24시간 코스피 대시보드".

### Goals

1. **종가 기준 변동률** — KRX 전일 종가 대비 현재 가격 변동 % 를 PRIMARY 변동률로 표시.
2. **USDT 환산 가치** — PRIMARY 가격 아래 1줄 sub-label 에 `≈ $X.XX USDT`.
3. **동적 PRIMARY** — 국장 장중 → KRX 실시간 메인 / 국장 마감 → HL→KRW 메인. 컨텍스트 뱃지·라벨로 swap 명시.
4. **초보자 친화 UX**:
   - 한글 라벨화 (PREMIUM/COOL/WATCH 등)
   - 용어 풀이 툴팁 (4개: 프리미엄/Z-스코어/HL/KRX)
   - 장 개장/마감 카운트다운
   - 색·기호 범례 (collapsible)
   - 페이지 1줄 인트로 (헤더 부제목 변경)
   - 모바일 카드 단순화

### Non-Goals

- 차트/스파크라인/이력 추가 ❌ (v2 backlog)
- 새 종목 추가 ❌
- 다른 거래소 source 추가 ❌
- 알림/Telegram 통합 ❌
- 모바일 PWA / 오프라인 모드 ❌
- v0.3.0 의 Bento 3-up 위계 변경 ❌ (유지)

### 카파시 / AI slop 가드

- 한글 라벨은 inline string 매핑 — 별도 i18n 시스템 도입 ❌
- 툴팁은 4개만 (프리미엄/Z/HL/KRX) — 6+ 개로 늘리지 말 것
- 카운트다운은 서버 계산값 1개 (`krxMinsUntilOpen` 또는 `krxMinsUntilClose`) — 클라이언트 setInterval refresh 없이 5초 SWR 갱신만으로 충분
- 범례 collapsible — 별도 modal/route 만들지 말 것
- "전체 보기" 토글 같은 사용자 선호 저장 (localStorage 추가) ❌ v2 backlog
- KRX 전일 종가 source 선택은 구현 단계 위임 (Naver/Yahoo/계산 중 가장 신뢰 가능한 source)

## 2. Data Flow Changes

### 2.1 Server-side (`server/lib/sources/naver.ts` + 어쩌면 `yahoo.ts`)

`PricePoint` 옵셔널 필드 추가:

```ts
type PricePoint = {
  // ... 기존 ...
  previousClose?: number;  // 신규 — KRX 전일 종가 (Naver-only, optional)
};
```

**KRX 전일 종가 source 우선순위 (B2 반영)**:

1. **1순위 — Naver** (장중/마감 모두): 별도 endpoint 또는 응답 내 `closePrice` 분석. 한국 시장 정확도 최고.
2. **2순위 — Yahoo `previousClose`** (`005930.KS` 등): 실패/차단 시 fallback.
3. **3순위 — 서버 last-good 메모리 캐시**: 1, 2 둘 다 실패 시 마지막 성공값 사용 (server-side 메모리, 4초 단일 flight 외 별도 24h TTL).
4. **모두 실패**: `previousClose` 미설정 → 클라이언트 `종가 변동률` = `"—"` + 라벨 `"종가 데이터 없음"` + tooltip `"전일 종가 데이터를 불러올 수 없습니다. KRX 24h 변동률로 fallback."` + **24h 변동률 표시로 자동 폴백** (UX 우선 — 정보 0보다는 24h 라도).

서버 응답에 `previousCloseSource: 'naver' | 'yahoo' | 'cache' | undefined` 필드 추가하여 클라이언트가 stale 여부 알 수 있게 함.

구현 시점에 실제 Naver / Yahoo 응답 확인 후 신뢰 가능한 source 결정.

### 2.2 Server-side (`server/lib/session.ts`)

`SessionState` 옵셔널 필드 추가:

```ts
type SessionState = {
  // ... 기존 ...
  krxMinsUntilOpen?: number;   // 다음 KRX 개장까지 분 (장 마감 중일 때만 set)
  krxMinsUntilClose?: number;  // KRX 마감까지 분 (장중일 때만 set)
};
```

기존 `calendar.ts` (KRX holiday) + `session.ts` (DST + 시간 계산) 재사용. 휴일이면 다음 개장일 09:00 까지 계산.

### 2.3 Client-side

위 두 필드를 새 컴포넌트가 사용. SWR 5초 갱신으로 카운트다운 자연스럽게 업데이트 (별도 setInterval 불필요).

### 2.4 Schema Versioning 정책 (Codex ⚠️ #8 반영)

- `PricePoint.previousClose`, `PricePoint.previousCloseSource`, `SessionState.krxMinsUntilOpen`, `SessionState.krxMinsUntilClose` 모두 **optional** → backward-compatible.
- `schemaVersion` (현재 1) **bump 안 함**. optional 필드 추가는 동일 schemaVersion 내 허용.
- 클라이언트: `previousClose === undefined` 폴백 동작 (§2.1 4번) 필수 단위 테스트.
- 향후 breaking change (필드 제거 / 의미 변경) 발생 시에만 schemaVersion 2 로 bump.

### 3.2 동적 PRIMARY swap 규칙 (Codex B1/⚠️ type 반영)

```ts
type Primary = {
  value: number;
  unit: 'KRW' | 'USD';
  source: 'krx' | 'hl';
  isStale?: boolean;  // KRX stale mid-session 시 true
};

function pickPrimary(args: {
  session: SessionState;
  hl?: PricePoint;
  krx?: PricePoint;
  fx?: FxRates;
}): {
  primary: Primary | null;          // ⚠️ null 가능
  secondary: Primary | null;
  contextLabel: string;
};
```

규칙 (우선순위 순):

| Case | 조건 | primary | secondary | contextLabel |
|---|---|---|---|---|
| C1 | `krx === 'ok'` && `session.krx === true` | KRX 실시간 | HL→KRW (fx 있으면) | "KRX 실시간 / HL 야간 발견가" |
| C2 | `krx === 'stale'` && `session.krx === true` (**B1 fix**) | HL→KRW (fx 있으면) **또는** KRX stale 값 (isStale=true) | (반대편) | "KRX 데이터 지연 — HL 임시 사용" |
| C3 | `session.krx === false` && HL ok && fx ok | HL→KRW | KRX 전일 종가 (있으면) | "HL 야간 발견가 / KRX 전일 종가" |
| C4 | `session.krx === false` && HL ok && fx 0 | HL USD native | KRX (있으면) | "환율 없음 — USD 표시" |
| C5 | HL 없음 && KRX 있음 | KRX (현재 시점 status 그대로) | null | "HL 가격 없음" |
| C6 | 둘 다 없음 | **null** | null | "데이터 없음" |

뱃지는 §3.5 의 `getStateBadge()` 가 별도 결정 — primary 선택과 분리.

### 3.3 종가 기준 변동률 계산 (Codex ⚠️ divide-by-zero 반영)

```ts
function pctVsClose(primaryKrw: number | null, previousClose: number | undefined): number | null {
  if (primaryKrw === null || previousClose === undefined || previousClose <= 0) return null;
  //                                                          ^^^^^^^^^^^^^^^^ divide-by-zero guard
  return ((primaryKrw - previousClose) / previousClose) * 100;
}
```

- 장중: `primaryKrw = krx.price` → KRX 실시간 vs 전일 종가
- 마감: `primaryKrw = hl.price * fx.usdtKrw` → HL 야간 발견가 vs 전일 종가
- `previousClose` 없음 → `—` + "종가 데이터 없음" 라벨

시각:
```
▲ +1.23%  종가 대비    (color: var(--up))
▼ -2.45%  종가 대비    (color: var(--down))
—  0.00%  종가 대비    (color: var(--text-dim))
—         종가 데이터 없음   (모든 dim)
```

### 3.5 State Badge 매핑 (Codex B3 반영)

```ts
type StateBadge = 'open' | 'night' | 'holiday';

function getStateBadge(session: SessionState): StateBadge {
  if (session.krx) return 'open';
  // !session.krx 분기:
  if (session.krxAfter || session.krxNight) return 'night';
  // 주말 + 한국 공휴일 (= calendar.ts isKrxHoliday)
  return 'holiday';
}
```

- `'open'` → 🟢 장중 (09:00-15:30 KST, 평일, 비공휴일)
- `'night'` → 🌙 야간 (마감 후 ~ 다음 개장 전, 평일 비공휴일)
- `'holiday'` → 🔴 휴장 (주말 + KRX 공휴일)

`labels.ts` 의 `STATE_BADGE_LABEL` 매핑 사용. App.tsx 의 인라인 카운트다운 chip (§5.3 참조) 안에서 동일 함수 호출.

### 5.2 용어 풀이 툴팁 (b) (Codex ⚠️ mobile 반영)

`src/components/InfoTip.tsx` 신규 — **hybrid: native title (desktop hover) + click-to-show inline popover (mobile/touch)**.

```tsx
type Props = { term: string; description: string; ariaLabel?: string };

export function InfoTip({ term, description, ariaLabel }: Props) {
  const [open, setOpen] = useState(false);
  // Click 으로 토글 (mobile tap-friendly), native title 은 desktop hover 보조
  return (
    <span className="info-tip-wrap">
      <button
        type="button"
        className="info-tip"
        title={description}
        aria-label={ariaLabel ?? `${term}: ${description}`}
        aria-expanded={open}
        onClick={() => setOpen((v) => !v)}
        onBlur={() => setOpen(false)}
      >
        ⓘ
      </button>
      {open && (
        <span className="info-tip-popover" role="tooltip">
          <strong>{term}</strong>: {description}
        </span>
      )}
    </span>
  );
}
```

CSS: popover는 absolute, `z-index: 100`, 카드 외곽 넘어가도 OK (max-width 280px). desktop hover 도 작동 (title 폴백). 1-tap to open, 1-tap-away to close (onBlur).

4개 위치 (사용자 미요청 항목 추가 금지):
- StockHeroCard 의 프리미엄 라벨 옆
- StockHeroCard 의 신호 라벨 옆
- StockHeroCard 헤더 ticker 옆 (KRX/HL 한 줄 설명)
- IndexCompactCard 의 "거래소 간 격차" 옆

용어 4개:

function formatMins(mins: number): string {
  if (mins < 60) return `${mins}분`;
  const h = Math.floor(mins / 60);
  const m = mins % 60;
  return m === 0 ? `${h}시간` : `${h}시간 ${m}분`;
}

// 헤더 JSX 안:
{(() => {
