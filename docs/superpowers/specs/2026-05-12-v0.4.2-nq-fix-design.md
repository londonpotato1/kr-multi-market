## 1. Context & Motivation

사용자 피드백: 브라우저 스크린샷에서 Nasdaq 100 카드 "—" (빈 가격). 진단 결과:

| Source | NQ symbol | 현재 상태 |
|---|---|---|
| HL xyz | (해당 없음) | 미상장 — `xyz` dex universe (74개) 에 NQ/NDX/QQQ 없음. JP225 / XYZ100 은 있음 |

### Goals (사용자 결정 옵션 A+B+E+F+Polygon+TwelveData)

1. **A — Yahoo Finnhub fallback**: 기존 `FINNHUB_TOKEN` 인프라 활용. Yahoo 영구 fix.
2. **B — Binance retry/backoff**: timeout / 429 시 1초 후 retry (1회). 단순 transient fail 흡수.
3. **E — Bybit linear QQQUSDT 추가**: 신규 source. Binance fail 대체.
4. **F — Bitget USDT-M futures QQQUSDT 추가**: 신규 source. 추가 redundancy.
5. **Polygon.io 옵셔널** (`POLYGON_API_KEY` env): 무료 5 calls/min — server cache 12s+ TTL 로 안전. paid 키 없으면 skip.
6. **TwelveData 옵셔널** (`TWELVEDATA_API_KEY` env): 무료 800/day. paid 키 없으면 skip.

### 2.4 Binance retry — `server/lib/sources/binance.ts` 수정 (Codex ⚠️ #5 반영)

```ts
async function fetchTickers24hrWithRetry(symbols): Promise<BinanceTicker24hr[]> {
  try {
    return await fetchTickers24hr(symbols);
  } catch (err) {
    // 429 시 Retry-After 헤더 우선 사용 (최대 5초 cap)
    let backoffMs = 1000;
    if (err instanceof Response && err.status === 429) {
      const retryAfter = parseInt(err.headers.get('Retry-After') ?? '1', 10);
      backoffMs = Math.min(retryAfter * 1000, 5000);
    }
    await new Promise(r => setTimeout(r, backoffMs));
    return await fetchTickers24hr(symbols);  // 1회 재시도
  }
}
```

**Latency budget**: timeout 5s (기존) + backoff 1~5s + 재시도 5s = **최대 ~11s** (worst case). source-cache TTL (Binance 1s) 가 이 latency 흡수 — 1초마다 새 cache miss 시 11s 응답 가능. 운영 acceptable (binance 가 평소 200ms 응답이므로 retry 트리거 자체가 rare).

### 3.2 IndexCompactCard headline fallback chain (Codex 🚨 #2 fix — option C)

**Alias 제거**. 각 source 가 `payload.bybit / payload.bitget / payload.polygon / payload.twelvedata` 슬롯 그대로 노출. VenueRow 가 자동으로 실제 source 라벨 ("BYBIT", "BITGET") 표시.

`IndexCompactCard.tsx` 변경 (headline + label):

```tsx
// 변경 후 (fallback chain + Yahoo 포함 + source label 표시 — Codex 🚨 E + ⚠️ B fix)
function pickHeadlineSource(
  payload: TickerPayload,
  fxAvailable: boolean,
  usdtKrw: number
): { value: number; source: string; label: string } | null {
  if (!fxAvailable) return null;
  // 우선순위: HL > Yahoo > Binance > Bybit > Bitget > Polygon > TwelveData
  // 모든 candidates 가 USD/USDT scale (§2.7 unit 정규화) — 안전한 * usdtKrw
  const candidates: Array<[string, PricePoint | undefined, string]> = [
    ['hl',         payload.hl,         'HL'],
    ['yahoo',      payload.yahoo,      'Yahoo'],
    ['binance',    payload.binance,    'Binance'],
    ['bybit',      payload.bybit,      'Bybit'],
    ['bitget',     payload.bitget,     'Bitget'],
    ['polygon',    payload.polygon,    'Polygon'],
    ['twelvedata', payload.twelvedata, 'TwelveData'],
  ];
  for (const [name, pp, label] of candidates) {
    if (pp && Number.isFinite(pp.price) && pp.price > 0) {
      return { value: pp.price * usdtKrw, source: name, label };
    }
  }
  return null;
}

// JSX: headline 옆에 small label
{headline && (
  <>
    <div className="index-compact-headline ts-index-headline">{fmtKrw(headline.value, 0)}</div>
    <div className="index-compact-usdt ts-subtitle">
      ≈ {fmtUsd(headline.value / usdtKrw)} USDT · 출처 {headline.label}
    </div>
  </>
)}
```

**Yahoo 포함 (Codex 🚨 E fix)**: §2.7 unit 정규화 후 yahoo 도 USD QQQ → safe.

**Source label 표시 (Codex ⚠️ B fix)**: headline 아래 USDT sub-label 에 "출처 Bybit" 등 micro label. Polygon/TwelveData 가 venue rows 에 없어도 사용자가 출처 확인 가능.

UI 영향: VenueRow 가 5개 (hl/yahoo/binance/bybit/bitget) 렌더 가능. Polygon/TwelveData 는 옵셔널이므로 카드에 표시 안 함 (env 활성화 시에만). NQ 카드는 venue 가 1개라도 있으면 정상 표시.

**spread pair / sourceHealth 도 실제 venue 그대로** — misrepresent 없음.

## 5. Risks

| # | Risk | Mitigation |
|---|---|---|
| R1 | Bybit/Bitget 한국 IP 차단 (KYC 정책) | 무료 public ticker endpoint — 인증 불필요. 차단 시 sourceHealth 자동 마킹 |

| R3 | TwelveData 800/day 초과 | TTL 120s 강제 (24h × 30/h = 720 calls/day < 800) |
| R4 | 신규 4 source → 응답 latency ↑ | 모두 Promise.all 병렬, 가장 느린 source 가 응답 시간 결정 |

| R6 | Finnhub free 60/min 초과 | yahoo cache TTL 5s × 60 = 720/h. Finnhub free 60/min = 3600/h. 안전 |
| R7 | Binance retry 가 진짜 outage 시 latency 2배 | retry 1회 + 1s 백오프 = 추가 ~2s. 허용 |
| R8 | Polygon TTL 12s + SWR 2s → 같은 Polygon 값 6번 연속 client 도착 | 가격 깜빡임 minimal (다른 source 값 fresh, headline source label "Polygon" 표시 시에만 stale). UI 영향 acceptable |
